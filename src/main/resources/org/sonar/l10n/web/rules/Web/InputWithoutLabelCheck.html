The <label> tag defines a label for an <input> element.

A label does not render as anything special, but will toggle the associated input when its text is clicked. It also improves the usability for visually impaired users: Screen readers will announce the label text whenever the focus is set on the input.

The for attribute of the <label> tag should be equal to the id attribute of the related <input> to bind them together.

This rule verifies that all <input> tags have an id attribute which is referenced by a label.

These input types are not verified by this rule:

  • submit
  • button
  • image
  • hidden

The following code snippet illustrates this rule:

<input type="text" name="a" />               <!-- Non-Compliant - no id -->
<input type="text" name="b" id="b" />        <!-- Non-Compliant - no matching label 'b' -->

<label for="c">...</label>
<input type="text" name="c" id="c" />        <!-- Compliant -->

<input type="hidden" name="d" value="...">   <!-- Compliant - 'hidden' type is excluded -->
<input type="submit" value="Send" />         <!-- Compliant - 'submit' type is excluded -->
