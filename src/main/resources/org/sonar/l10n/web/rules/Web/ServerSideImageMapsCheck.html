When the ismap attribute is present on a img tag, it is a server-side image map: The browser sends the coordinates of the clicked point to the server. For any person who cannot use a mouse, this form of navigation is inaccessible as it is only the position of the cursor on the image which will determine the action. Client-side image maps, which use the usemap attribute, on the other hand allow for each clickable area to specify an alternate text enabling accessibility. Moreover, in terms of separation of concerns, it is definitely better to leave the task of mapping pixels to links to the client.

The following code:

<a href="click_on_world_map.php" target="_self">
  <img src="world_map.png" ismap>                   <!-- Non-Compliant -->
</a>

should be refactored into:

<img src="world_map.png" usemap="#world_map">       <!-- Compliant -->

<map name="world_map">
  <area shape="rect" coords="0,0,10,10" href="france.html" alt="France">
  <area shape="circle" coords="20,20,10" href="spain.html" alt="Spain">
  <area shape="circle" coords="30,30,8" href="england.html" alt="England">
  <!-- ... -->
</map>
